# Page Bridge Session Hardening

Status: TODO

Importance: 8/10

Rank: 8

## What The Feature Is

Harden the direct page overlay bridge between the React UI and the page-world WebSocket engine.

The current bridge uses `window.postMessage` with source marker strings. That works for a cooperative page, but a hostile or noisy page can also post messages with the same markers.

## What To Add

- A per-injection session id or nonce generated by the extension and required on every bridge message.
- Narrow message schemas for connect, send, stop, status, and frame events.
- Bridge lifecycle states: injected, ready, stale, disconnected, and rejected.
- Explicit rejection logging for malformed or wrong-session messages.
- Cleanup when the overlay closes or the page unloads.
- Tests for valid message, wrong source marker, wrong nonce, malformed payload, and stale session behavior.

## How It Benefits Users

- Direct page overlay behavior becomes more predictable on complex sites.
- The tool reduces accidental or malicious cross-talk with page scripts.
- Runtime failures become easier to diagnose.
- Page-context testing remains visibly different from extension-context testing.

## Why This Importance

8/10 because page-world control is powerful and easy to get wrong. Hardening should happen before expanding page-context features.

## Contract Notes

- Source marker strings are labels, not authority.
- Do not broaden the bridge to support extra commands without a typed contract and tests.
- Prefer passive CDP capture when the user only needs observation.
